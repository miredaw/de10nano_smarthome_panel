GSM / SMS Module

SIM800L

OFFLINE
Interface Altera UART IP — direct MMIO via /dev/mem
LW Bridge Register 0xFF200080 (uart_sim800l_0, offset 0x80)
Baud Rate 9600 bps, 8N1
SMS Cooldown 300 s (SMS_COOLDOWN_SEC in smart_home.h)

Hardware — SIM800L EVB (Evaluation Board)

Board:SIM800L EVB with onboard LDO regulator
TX (FPGA→SIM800L):JP1 Pin 19 (FPGA PIN_D12) — 3.3V compatible, no divider needed
RX (SIM800L→FPGA):JP1 Pin 20 (FPGA PIN_AD20) — direct 3.3V
Power:5V input via external DC-DC buck converter (≥ 2A)
Onboard LDO:EVB steps down 5V → 4V for SIM800L chip
PWRKEY:Brief LOW pulse may be needed to power on if not auto-starting
SMS format:AT+CMGF=1 (text mode)
Send command:AT+CMGS="+39XXXXXXXXXX" → message → Ctrl-Z (0x1A)
Triggered by:Alarm bit 0→1 edge detection in handle_alarms()

Altera UART Register Map (base 0xFF200080)

+0x00 rxdata:[7:0] received byte (read when RRDY=1)
+0x04 txdata:[7:0] transmit byte (write when TRDY=1)
+0x08 status:bit7=RRDY, bit6=TRDY, bit5=TMT, bit3=ROE
+0x0C control:interrupt-enable bits (kept at 0)

Communication Architecture

End-to-end data flow

1. FPGA VHDL:Reads BME280 (I²C) + MCP3008 (SPI) + PIR (GPIO) every ~100ms → stores in Avalon registers
2. HPS C (main_supervisor.c):Polls Avalon registers every 1s → builds JSON payload → sends via AT+MQTTPUB every 5s
3. ESP32-WROOM-32:Receives AT+MQTTPUB command on UART1 → publishes to Mosquitto broker over WiFi/TCP
4. Mosquitto (VPS):Receives on smarthome/sensors + smarthome/alarms → forwards to Flask backend subscriber
5. Flask + SocketIO:MQTTHandler receives message → emits sensor_update / alarm_update to all browsers via SocketIO
6. Browser:socket.on('sensor_update') → updates DOM elements live, no page reload needed
Parallel: SIM800L:On alarm edge (0→1 bit transition) → AT+CMGS → SMS to configured phone number (300s cooldown)
